REQUISITOS:
- Docker Desktop (PostgreSQL e Ollama em containers)
- Python 3.10+ com pip

COMO EXECUTAR (todos os comandos a partir da pasta da entrega, onde estão os 8 arquivos):

1) Descompactar o protótipo:
   - Extrair 07.Prototipo.zip; será criada a pasta "prototipo/".

2) Subir o PostgreSQL e criar o banco:
   docker run -d --name postgres-copa -e POSTGRES_PASSWORD=copa123 -e POSTGRES_DB=copa_mundo -p 5432:5432 postgres:16

3) Carregar DDL e DML:
   docker cp 05.DDL.sql postgres-copa:/tmp/DDL.sql
   docker cp 06.DML.sql postgres-copa:/tmp/DML.sql
   docker exec -it postgres-copa psql -U postgres -d copa_mundo -f /tmp/DDL.sql
   docker exec -it postgres-copa psql -U postgres -d copa_mundo -f /tmp/DML.sql

4) Subir o Ollama e baixar o modelo:
   docker run -d -v ollama:/root/.ollama -p 11434:11434 --name ollama ollama/ollama
   docker exec -it ollama ollama pull gemma3:1b

5) Instalar dependências Python e rodar o protótipo:
   cd prototipo
   pip install -r requirements.txt
   python main.py

6) Dados de login no protótipo:
   Host: localhost   Porta: 5432   Banco: copa_mundo
   Usuário: postgres   Senha: copa123

OBSERVAÇÕES / O QUE NÃO FOI REALIZADO:
- O modelo gemma3:1b (1B parâmetros) tem limitações para gerar SQL a partir de
  linguagem natural; perguntas mais complexas podem retornar texto em vez de SQL.
  O protótipo detecta isso, exibe a resposta crua do modelo e cancela a execução.
- O SQL gerado pela IA sempre é exibido antes da execução, exigindo confirmação.
- A opção "Digitar SQL diretamente" aceita múltiplos comandos separados por ";".
